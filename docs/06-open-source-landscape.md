# Опенсорс по косилкам: что берём и откуда

Не пишем мозги с нуля — есть зрелые проекты. Ниже — что реально переиспользуем
под наш кейс (провод периметра + ESP32-S3 + HA/свой сервер), и лицензии.

## Карта проектов

| Проект | Навигация | Железо/MCU | Чем полезен НАМ | Лицензия |
|--------|-----------|------------|------------------|----------|
| **Ardumower — Azurit** | провод периметра + matched filter | Arduino (AVR/Due) | 🎯 ядро: приём провода (знак+величина), PID моторов, одометрия, IMU, **docking по проводу** | GPL |
| **Ardumower — Sunray** | RTK-GPS без провода | SAMD51/Due, есть ESP32-ветки | фаза 5+: RTK, планирование пути, fix-обработка | GPL |

## Что конкретно лифтим из Azurit (под ESP32-S3)

- `perimeter.*` — matched filter: корреляция сигнала катушки с кодом sender,
  знак (внутри/снаружи) + magnitude (дистанция). → лечит **кручение на месте**.
- `motor.*` — PID скорости колёс, плавный разгон/торможение.
- одометрия (Холлы) + IMU fusion — курс и путь.
- логика **docking по проводу** → лечит **промах мимо базы**.
- sender (передатчик в базу): код для генерации асимметричного псевдошум-сигнала
  в провод (если штатный sender некодированный — ставим свой на ESP/Arduino).

Адаптация: AVR-специфику (таймеры/ADC) меняем на ESP32-S3 (`adc_continuous`/I2S-ADC
+ DMA для выборки катушки, аппаратные PWM `ledc`, прерывания для Холлов, FreeRTOS-задачи).

## Что лифтим из OpenMower (на сервер, не на ESP32)

- модель **карты участка и зон** (полигоны, no-go зоны);
- идея **планировщика покрытия** (boustrophedon / по полосам) — пока как «мягкие»
  зоны поверх провода, для аналитики «где давно не косили»;
- структура телеметрии/состояний для дашборда.

OpenMower целиком брать не будем: он на Raspberry Pi + ROS + конкретное железо
YardForce. Нам нужен лёгкий ESP32-стек, поэтому берём *идеи и структуры данных*, не код.

## Лицензии — важно

- Ardumower (Azurit/Sunray) — **GPL**. Если будем публиковать свою прошивку,
  производную от их кода → она тоже GPL, исходники открыты. Для домашнего/личного
  использования ограничений нет.
- OpenMower — смотреть лицензии по компонентам репозитория перед копированием кода.
- Идеи/алгоритмы (matched filter, PID, boustrophedon) — не патентно-чисты по
  умолчанию: в патентах Husqvarna есть docking/navigation методы. Для личного
  использования ок; для продукта — отдельная проверка.

## Практический план переиспользования

1. Поднять Azurit в исходниках, изолировать `perimeter`/`motor`/одометрию/docking.
2. Сделать тонкий HAL-слой ESP32-S3 (ADC-DMA, ledc-PWM, I2C-IMU, Холлы) под эти модули.
3. Сверху — наш сетевой слой (MQTT в HA + свой сервер, OTA, расписание).
4. Серверный планировщик/карту — по мотивам OpenMower.

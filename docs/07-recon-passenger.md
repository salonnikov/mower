# Фаза 0.5 — «Пассажир-разведчик» (неинвазивно, без вскрытия)

Идея: косилку **не трогаем**, она косит как раньше. Сверху ставим автономный
модуль на ESP32, который только **слушает** и пишет логи. На реальных данных
отлаживаем приём провода (matched filter) и модель движения — до любого вмешательства.

Все сенсоры разведчика **переходят потом в настоящий мозг** — это не выброшенная работа.

## Зачем (что снимаем без риска)

| Хотим узнать | Чем меряем | Что даёт |
|---|---|---|
| Кодированный ли родной сигнал провода, частота, форма | **катушка-антенна + ОУ → ADC** | ответ «менять ли sender» БЕЗ вскрытия; модель для matched filter |
| Как падает поле у границы | катушка (зависимость величины от расстояния) | настройка порогов wire-following |
| Как робот реально поворачивает/отскакивает от провода | **IMU (гиро+акс)** | модель поведения, калибровка курса |
| Сигнатура «кручения на месте» | IMU: продолжительный большой `turn_rate` | поймать баг в реальных данных и сопоставить с сигналом провода |

## Состав модуля-разведчика

- **ESP32** (любая из твоих плат) — 2 ядра: ядро 0 сбор, ядро 1 Wi-Fi.
- **IMU**: BNO085 (или MPU6050 на первое время) по I²C — гиро+акселерометр.
- **Катушка-антенна** (несколько витков, диаметр ~5–10 см) + ОУ-фронтенд (LM358/MCP6002,
  смещение Vcc/2) → вход ADC1 (GPIO36). Ловит поле родного провода.
- **microSD** (SPI) — сырой высокочастотный лог катушки (Wi-Fi столько не вывезет);
  + Wi-Fi/MQTT для live-сводок.
- **Своё питание**: маленький LiPo + buck/LDO. Модуль полностью автономен —
  в электронику косилки НЕ влезаем (это и есть «неинвазивно»).
- **GPS** (опц., NEO-M8N) — привязка трека на улице, контекст.
- Крепление: 3D-печатный лоток/липучка на корпус. Катушку — наружу/вниз, ближе к земле.

## Что логируем (схема записи)

```
SD (быстро, локально):   катушка-ADC, сырые семплы + таймштампы
MQTT (медленно, сводки): turn_rate, tilt, wire_freq, wire_mag, инциденты
```
Топики (в общий контракт, см. docs/01):
`mower/mi302/recon/turn_rate`, `/recon/tilt`, `/recon/wire_freq`, `/recon/wire_mag`,
`/recon/event` (напр. `spin_suspected`, `wire_cross`).

## Как анализируем (после нескольких сессий кошения)

1. **Сигнал провода** → FFT/корреляция: частота, период кода, симметричен ли
   (если симметричен — знак внутри/снаружи не вытащить, sender менять обязательно).
2. **Эпизоды «кручения»** (продолжительный большой `turn_rate`) → сопоставить с
   сигналом провода в тот момент: понять, почему родной мозг теряет направление
   (подтвердить гипотезу про бедный сигнал).
3. **Поведение у границы** → как часто/под каким углом отскок, где застревает.

## Связь с дальнейшими фазами

- Катушка+ОУ, IMU и код matched filter — **переезжают в мозг** (фазы 2–5).
- Курс/скорость в мозге берём с **одометрии (Холлы) + IMU**.
- Замена sender'а на ESP8266 (кодированный сигнал) — делаем ПОСЛЕ того, как
  разведчик подтвердит, каков родной сигнал (docs/06, sender).

## Прошивка разведчика

Отдельный проект: `firmware/recon/` (см. README там). Логгер на ESP32
(C++/PlatformIO), не трогает прошивку мозга. ESPHome тут не подходит — нужен
быстрый ADC + FFT/корреляция + запись на SD.

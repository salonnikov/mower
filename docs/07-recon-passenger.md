# Фаза 0.5 — «Пассажир-разведчик» (неинвазивно, без вскрытия)

Идея: косилку **не трогаем**, она косит как раньше. Сверху ставим автономный
модуль на ESP32, который только **слушает** мир и пишет логи. На реальных данных
отлаживаем приём провода (matched filter) и модель движения — до любого вмешательства.

Все сенсоры разведчика **переходят потом в настоящий мозг** — это не выброшенная работа.

## Зачем (что снимаем без риска)

| Хотим узнать | Чем меряем | Что даёт |
|---|---|---|
| Кодированный ли родной сигнал провода, частота, форма | **катушка-антенна + ОУ → ADC** | ответ «менять ли sender» БЕЗ вскрытия; модель для matched filter |
| Реальная скорость по земле (без проскальзывания колёс) | **доплер-радар** | ground-truth для одометрии, детект пробуксовки |
| Сигнатура «кручения на месте» | радар ≈ 0 вперёд + гироскоп показывает вращение | поймать баг в реальных данных |
| Как робот реально поворачивает/отскакивает от провода | **IMU (гиро+акс)** | модель поведения, калибровка курса |
| Где идёт провод и как падает поле у границы | катушка (зависимость величины от расстояния) | настройка порогов wire-following |

## Состав модуля-разведчика

- **ESP32** (любая из твоих плат) — мозг логгера, 2 ядра: ядро 0 сбор, ядро 1 Wi-Fi.
- **IMU**: BNO085 (или MPU6050 на первое время) по I²C — гиро+акселерометр.
- **Доплер-радар**: см. раздел ниже. У тебя есть пара разных — заведём оба, сравним.
- **Катушка-антенна** (несколько витков, диаметр ~5–10 см) + ОУ-фронтенд (LM358/MCP6002,
  смещение Vcc/2) → вход ADC1 (GPIO36). Ловит поле родного провода.
- **microSD** (SPI) — сырой высокочастотный лог (радар IF и катушка пишутся быстро,
  Wi-Fi столько не вывезет). + Wi-Fi/MQTT для live-сводок.
- **Своё питание**: маленький LiPo + buck/LDO. Модуль полностью автономен —
  в электронику косилки НЕ влезаем (это и есть «неинвазивно»).
- **GPS** (опц., NEO-M8N) — привязка трека на улице, контекст.
- Крепление: 3D-печатный лоток/липучка на корпус. Радар и катушку — наружу/вниз.

## Доплер-радар: какой и как считать скорость

Доплер: `f_d = 2·v / λ`, где `λ = c/f`.
- **HB100 (10.525 ГГц):** λ≈28.5 мм → **≈70 Гц на 1 м/с** (= 19.5 Гц на км/ч).
- **CDM324 (24.125 ГГц):** λ≈12.4 мм → **≈161 Гц на 1 м/с** — вдвое лучше разрешение.

Косилка едет медленно (~0.3 м/с ≈ 1 км/ч):
- HB100 → ~21 Гц, CDM324 → ~48 Гц. **Оба ловятся**, но на низкой скорости
  **24 ГГц (CDM324) предпочтительнее** — частота выше, разрешение лучше.

Тонкости:
- Выход IF радара — микровольты–милливольты: нужен **усилитель** (×100–1000, полоса
  ~5–500 Гц) перед ADC. Без него ESP32 ничего не увидит.
- Скорость считаем по **частоте** сигнала (zero-crossing или FFT на ESP32), а не по амплитуде.
- Радар видит модуль скорости, не знак (вперёд/назад) — знак берём из гироскопа/контекста.
- На «кручении на месте» вперёд-скорость ≈ 0, а гиро крутится → ловим сигнатуру бага.

## Что логируем (схема записи)

```
SD (быстро, локально):     катушка-ADC, радар-IF, сырые семплы + таймштампы
MQTT (медленно, сводки):   speed_mps, turn_rate, tilt, wire_freq, wire_mag, инциденты
```
Топики (в общий контракт, см. docs/01):
`mower/mi302/recon/speed`, `/recon/turn_rate`, `/recon/wire_freq`, `/recon/wire_mag`,
`/recon/event` (напр. `spin_detected`, `wire_cross`).

## Как анализируем (после нескольких сессий кошения)

1. **Сигнал провода** → FFT/корреляция: частота, период кода, симметричен ли
   (если симметричен — знак не вытащить, sender менять обязательно).
2. **Скорость (радар) vs обороты колёс** (когда позже подключим Холлы) → коэффициент
   проскальзывания, калибровка одометрии.
3. **Эпизоды «кручения»** → сопоставить с сигналом провода в тот момент: понять,
   почему родной мозг теряет направление (подтвердить гипотезу про бедный сигнал).
4. **Поведение у границы** → как часто/под каким углом отскок, где застревает.

## Связь с дальнейшими фазами

- Катушка+ОУ, IMU, радар, код matched filter — **переезжают в мозг** (фазы 2–5).
- Радар остаётся как slip-free датчик скорости — это лучше, чем одна одометрия.
- Замена sender'а на ESP8266 (кодированный сигнал) — делаем ПОСЛЕ того, как
  разведчик подтвердит, каков родной сигнал (docs/06, sender).

## Прошивка разведчика

(C++/PlatformIO), не трогает прошивку мозга. ESPHome тут не подходит — нужен
быстрый ADC + FFT/корреляция + запись на SD.

# memory-map.md — карта памяти и периферии (chip1, GD32F305, Cortex-M4F)

Легенда: `✔` — механически подтверждено (вывод инструмента / datasheet-факт / живой замер); `ⓗ` — гипотеза о РОЛИ, проверить на живом чипе.

Метод: базовые адреса периферии извлечены **механически** из литерал-пулов приложения — счёт обращений `grep '=0x4[0-9a-f]{7}'` по полному дизасму `pD 0xe8000 @ 0x08018000`. Соответствие «база → периферия» — **факт из datasheet GD32F30x** (`✔`), поэтому идентичность регистра доверяема. Что помечено `ⓗ` — это **функциональная роль** (какой USART = дисплей и т.п.), она из контекста, не из карты регистров.

---

## 1. Флеш (внутренняя, 1 МБ, load base 0x08000000) `✔`

| Диапазон | Что | Опора |
|---|---|---|
| `0x08000000–0x08018000` | **Загрузчик (BTL)**, 96 КБ | `✔` reset `fcn.08011a3c` здесь; MB@0x18000 |
| `0x08018000–0x08100000` | **Приложение (MB)**, ≤ `0xE8000` (928 КБ) | `✔` CRC-гейт `fcn.08006fe8` над `0x08018000` |

Внешняя SPI-флеш: env EasyFlash + staging-области OTA (`ⓗ` контекст, см. reverse §9).

---

## 2. RAM (SRAM, 0x20000000) — подтверждённые анкеры `✔`

| Адрес | Что | Статус |
|---|---|---|
| `0x200000bc` | **main state** (0/2/3=standby/5=движение) | `✔` живой SWD + `set_main_state` пишет сюда |
| `0x200000ac` | база блока состояния (`+0x10`→state) | `✔` литерал в `fcn.0807976c` |
| `0x2000be28` | **одометрия левого колеса** (тикает при езде) | `✔` живой SWD; писатель `ⓗ` (см. `decompiled/odometry.md`) |
| `0x2000be58` | **одометрия правого** (шаг +0x30 от лев.) | `✔` живой SWD |
| `0x2000b250` | кандидат мотор-значения (~1037) | `ⓗ` живой замер, привязка к коду не доказана |
| `*0x20000338` | указатель на **movebase-объект** (валиден в активных состояниях) | `✔` живой SWD; 23 обращения в коде |

## 3. RAM-глобалы (низкая область — указатели на объекты/сервисы)

Извлечено механически: топ баз `grep '=0x2000[0-9a-f]{4}'` по дизасму (число = кол-во обращений). Роли — `ⓗ` из reverse-доков (перепроверять):

| База | Обращений | Роль `ⓗ` |
|---|---:|---|
| `0x20000168` | 149 | горячий глобал (лог/таймбаза?) — не установлено |
| `0x20000330` | 50 | объект — не установлено |
| `0x200002f0` | 48 | объект — не установлено |
| `0x20000350` | 47 | **данные BMS** (reverse §5) |
| `0x20000338` | 23 | **movebase-объект** `✔`(живой) |
| `0x20000098` | 35 | структура докинга (reverse §4) |
| `0x200000a0` | — | ctx зарядки (reverse §5) |
| `0x200000a8` | — | process-manager (reverse §1) |
| `0x200003b0` | 29 | screen-event для UI-машины (reverse §7) |
| `0x2000027c` | — | загруженный user password (reverse §8) |
| `0x200000ae` | — | флаг низкого заряда (reverse §5) |

> Высокая RAM (`0x2000bXXX`, `0x2000eXXX`) в литерал-пулах **не встречается** — это буферы/heap-объекты, адресуемые через указатели из низких глобалов. Поэтому одометрия/коилы/мотор ищутся через объект, а не по абсолютному адресу (см. `decompiled/odometry.md` §1).

## 4. Периферия (база `✔` datasheet, роль `ⓗ`)

Число = кол-во обращений в приложении (индикатор «насколько активно используется»).

### AHB
| База | Периферия `✔` | Обр. | Роль `ⓗ` |
|---|---|---:|---|
| `0x40020000` | **DMA0** | 17 | ADC/USART DMA `ⓗ` |
| `0x40020400` | DMA1 | 7 | `ⓗ` |
| `0x40021000` | **RCU** | 14 | тактирование |
| `0x40022000` | **FMC** | 10 | контроллер флеши (запись патча) |
| `0x40023000` | **CRC** | 3 | **HW CRC-32 `✔`** (гейт прошивки, reverse §9) |

`ⓗ` Заметки по ролям (требуют живого подтверждения):
- Четыре USART (USART0/1/2, UART3) → три JSON-линка (dpport/bdport/ledport) + один лог/RTT. Какой физически = дисплей — сверить по пинам/трафику.
- IMU (BNO055 + TDK ICM-42688 в драйверах, см. `decompiled/imu.md`): BNO055 обычно на I2C1 (`0x40005800`), TDK ICM-42688 — на SPI. Активный интерфейс определить живьём.
- Одометрия: general-purpose TIMER (1/2/3/4) в encoder-mode — правдоподобный механизм счёта холлов; подтвердить watchpoint’ом на `0x2000be28` и чтением `TIMERx_CNT` (см. `decompiled/odometry.md` §5).
- Моторы (колёса/нож): advanced TIMER0/TIMER7 для комплементарного PWM BLDC + возможно SPI0 к драйверу — `ⓗ`.

## 5. Как воспроизвести извлечение
```
sudo docker run --rm -u root -v /opt/mower:/work -w /work radare/radare2 \
  r2 -q -p /work/tools/r2out/gd32.r2proj \
  -c 'e scr.color=0; e asm.bytes=false; s 0x08018000; pD 0xe8000' 2>/dev/null > app.asm
grep -oE '=0x4[0-9a-f]{7}' app.asm | sort | uniq -c | sort -rn   # периферия
grep -oE '=0x2000[0-9a-f]{4}' app.asm | sort | uniq -c | sort -rn # RAM-глобалы
```
(проект грузится БЕЗ бинаря в аргументах — путь зашит внутри).

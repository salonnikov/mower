# Подсистема сбора сигнала коилов (border/провод) и ADC — chip1 (GD32F305)

Дамп: `/opt/mower/dist/gd32-mainboard-dump-v1.bin` (Cortex-M4 Thumb-2). Бинарь двойного
маппинга: реальный код по `0x0800xxxx`. Декомпиляция — Ghidra через radare2 `pdg`
(механический вывод, приведён verbatim в блоках ```c). Интерпретация помечена
`ⓗ (гипотеза, проверить на живом чипе)` либо `✔ (подтверждено строкой/xref/литералом)`.

Легенда полей контекста (border service):
- `ctx = *(*(root+4) + 0x24)` — узел границы; `*(ctx+0x24)` = состояние ЛЕВОЙ катушки,
  `*(ctx+0x2c)` = состояние ПРАВОЙ катушки. Значения дискретны: `0` / `1` / `2`.

  выборки `SAMPT1/SAMPT0` (offset `0x10/0xc`, по 3 бита/канал). Подтверждает: 0x40012400 = ADC0.

Доказательства ADC:
- ✔ база `0x40012400` = ADC0 (лит. `0x4001244c - 0x4c`); `RDATA=0x4001244c` — источник DMA;
  `0x40020000` = DMA0 (контроллер). RAM-приёмники: `*(0x200005dc)+0x14` (5-кан.) и
  `*(0x20000620)+0x14` (8-кан.) — указатели на буфер (в дампе `0xffffffff`, т.е. до init).
- ✔ каналы: 5-кан. {15,14,16,9,17}, 8-кан. {15,14,16,9,17,11,12,13}. Каналы 16/17 —
  внутренние (темп./Vrefint) GD32; внешние 9,11,12,13,14,15 — сенсоры платы.
- вызовы: `fcn.080217e8` из `0x8059b98`, `fcn.08021914` из `0x8059e38` (общий init-диспетчер).
- `fcn.080116a4`: `ldr r0,[0x40012400]; bl 0x80063d8; movw r0,0x609; bl 0x8010cb4` —
  сброс/включение тактирования ADC0 (RCU), т.е. приведение ADC0 в рабочее состояние. ✔

Интерпретация ADC:
- ⓗ Это сканирующий ADC0 c DMA0: непрерывно оцифровывает набор каналов в кольцевой RAM-буфер.
  Сырьё катушек (провод-периметр) — среди внешних каналов {9,11,12,13,14,15}; конкретную пару
  «левая/правая катушка» надо определить живьём (подать сигнал на катушку, смотреть, какой
  индекс буфера меняется). Магнитуды L/R для 0x0804b0b8 (`+4`,`+8`) — вероятно производные от
  этих RAW-отсчётов (после выпрямления/усреднения на плате сигнала или в border-сервисе).
- ⓗ ЖИВОЙ АНКЕР: по SWD видели 2 меняющихся канала около `0x2000eae0`. RAM-приёмник DMA задаётся
  указателями `*(0x200005dc)` / `*(0x20000620)` (в статике не инициализированы). Не исключено,
  что буфер аллоцируется в район `0x2000exxx` — но статикой это НЕ доказано. Проверить: прочитать
  `*(0x200005dc)` и `*(0x20000620)` на живом чипе → получить реальный адрес RAW-буфера ADC.

## Итоговая цепочка: катушка → ADC → флаг 0/1/2 → навигация

1. Аналоговый сигнал катушек (провод-периметр) поступает на входы ADC0 (`0x40012400`).
   `fcn.080217e8`/`fcn.08021914` настраивают сканирование каналов {9,11,12,13,14,15(+16,17)} с
   DMA0 в RAM-буфер (`*(0x200005dc)+0x14` / `*(0x20000620)+0x14`). ✔ периферия, ⓗ какой канал = катушка.
2. Из RAW-отсчётов формируются магнитуды L/R и флаги валидности; они читаются как поля объекта
   border-signal: `fcn.0805034c()` → метод `vtable+0x10` → `{+4:magL, +8:magR, +0xc:okL, +0x10:okR, +0x14:state}`. ⓗ
3. Порог близости к станции: `fcn.0804b0b8` — near-station при обеих магнитудах `<= 31000`
   (лит. `0x7918`) и валидных флагах. ✔
4. Дискретное состояние стороны хранится в контексте узла границы: `*(ctx+0x24)` = ЛЕВО,
   `*(ctx+0x2c)` = ПРАВО (значения 0/1/2). Сырой бит стороны даёт `fcn.0804f2ee` (1 или 2). ✔
5. `fcn.0802c424` — резолвер: из пары (ЛЕВО,ПРАВО) + режим `+0x12a` собирает единый флаг 0/1/2
   в `*param_2`. `fcn.0802c900` — на переходе состояния выдаёт этот флаг наружу через callback
   `vtable+0x88` в навигацию/planner. ✔
6. Отдельная плата сигнала общается кадрами; `fcn.08063880` (bdport) диспетчеризует входящие
   кадры по top-байту типа на до 5 обработчиков. ✔

## Открытые ⓗ (проверить на живом чипе)

- Какой именно ADC-канал соответствует левой и правой катушке (внешние 9/11/12/13/14/15).
- Реальный адрес RAW-буфера DMA: прочитать `*(0x200005dc)` и `*(0x20000620)`; сверить с
  наблюдаемым регионом `~0x2000eae0`.
- Где именно RAW→магнитуда (выпрямление/детект амплитуды): на плате сигнала или в прошивке
  chip1 (кандидат — потребители `fcn.0805034c`/метод `+0x10`).
- Семантика дискрета: `1` = «внутри», `2` = «снаружи», `0` = «нет сигнала» — подтвердить
  подачей сигнала и чтением `*param_2`/callback `+0x88`.
- Смысл инверсии `*(ctx+0x72)==1` в 0x0802c424 (полярность/направление намотки провода).

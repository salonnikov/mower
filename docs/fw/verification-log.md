# Журнал независимой проверки (аудит + самопроверка)

Задача заказчика: «не доверяй прошлой декомпиляции, проверь как независимый разработчик». Метод: свой инструментарий (radare2 в docker `radare/radare2`, прямое чтение байт python), **без опоры на прошлый r2-проект**. `✔`=воспроизведено механически.

|---|---|---|

| chip2=GD32F30x 256КБ, reset `0x080001b4` | SP `0x20000988`, reset `0x080001b4` | `✔` |

**Найденная неточность прошлой доки:** вызов `rand(0x2d,0x7d)` привязан к «`fcn.08044188`», а физически он в **под-функции** докинга (`0x0803fd96` < старта `0x08044188`). Суть верна, адрес вольный. На выводы не влияет.

**Итог аудита:** грубых косяков нет, прошлый реверс достоверен как карта.

## B. Самопроверка МОИХ находок (эта сессия)

**Сырые байты (чтение напрямую, не дизасм):** все совпали —
`set_main_state[0x0807976c]=0b28`(cmp r0,0xb) ✔; `crc8[0x080241d6]=8c24`(movs r4,0x8c) ✔; `dock[0x0803fd96/98]=7d21/2d20`(0x7d,0x2d) ✔; `lmotor pwm[0x0805f9ac]=40f2fd50`(movw 0x5fd=1533) ✔.

**Сверка адресов с afl (1783 фн):** `0x0801f4dc`(50б) ✔, `0x0807976c`(10б) ✔, `0x080241d0`(44б) ✔, `0x0800214c`(462б) ✔, `0x0804f2c8`(38б) ✔, `0x0804f350` rain(34б) ✔, `0x08044188`(15666б) ✔.

**Пойманные СВОИ огрехи (честно):**
1. Баг в моём grep самопроверки (лишний ноль `0x00805f620`) → ложные «не найдена». Исправлено, пересверено.
2. Входы приводов `left/right_motor_rotation` (`0x0805f620`/`0x0806fce0`): логика подтверждена (дизасм+байты), но afl **не выделил их как функции** — они вызываются **косвенно через vtable драйвера** (`blx` по указателю из сервис-объекта `lmdrv`/`rmdrv`), поэтому статический граф вызовов их не видит. → **правильный способ звать их из нашего кода — брать указатель из зарегистрированного vtable драйвера, а не по жёсткому адресу** (переживёт пересборку и смену версии). Точный офсет указателя в объекте — досверить.

## C. Что это меняет для плана
Объектная архитектура (сервисы с vtable, косвенные вызовы) означает: наш модуль должен дергать драйверы **через их зарегистрированные объекты/vtable**, а не по абсолютным адресам. Это надёжнее и это же — естественная точка инъекции (WP3): подменить/обернуть указатели в сервис-объектах process-manager, сохранив драйверный слой.

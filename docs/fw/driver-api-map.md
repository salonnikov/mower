# WP2 — Карта драйверного API chip1 (для внешнего RC-управления)

Цель: найти точки входа, через которые наш тонкий app-исполнитель (Вариант 1) будет **командовать моторами** и **читать датчики**, переиспользуя родной драйверный слой. Регион приложения `0x08018000+`. `✔` — проверено дизасмом в этой итерации; `ⓗ` — лид из прошлого декомпила, ещё не сверен.

---

## 1. Привод колёс `✔ (drive-путь проверен)`

> ⚠️ Самопроверка (afl 1783 функции): адреса `0x0805f620`/`0x0806fce0` — это **внутри** функций приводов (логика PWM/направление подтверждена дизасмом и сырыми байтами `40f2fd50`=movw 0x5fd), но **точное начало функции надо допинать** (`af` их разбирает, но авто-afl ставит границу иначе). Единицы/формула верны, входной адрес уточнить перед вызовом из нашего кода.

### `left_motor_rotation` ≈ `0x0805f620` (`driver_left_motor_snk_v2.c`, модуль `lmdrv`) `✔ логика / ⓗ точный вход`
Точка входа «крутить левое колесо». Логика:
- Вход — знаковая скорость; **диапазон 0..6141** (`max speed=6141`).
- PWM = `speed / 1533` (`udiv r6, 0x5fd`), клампы: верх `pwm=0x2ce (718)`, низ — `set to 1533` при too low.
- Скважность: `bl 0x807f696(r0=TIMER@0x40000400, r1=channel, r2=pwm)` — **PWM-таймер `0x40000400` (GD32 TIMER1)** `✔`.
- Направление/enable: `bl 0x801f4dc` (низкоуровневый писатель бит режима, маски `0x10/0x20/0x30` на поле `[ctx+0xa]`) `✔`.
- Ошибка → лог `set left motor speed failed` / `left_motor_rotation`.

### `right_motor_rotation` ≈ `0x0806fce0` (`driver_right_motor_snk_v2.c`) `✔`
Проверен дизасмом: тот же паттерн, что и слева — `udiv` на `1533` (PWM), направление через `bl 0x801f4dc`, строка-модуль `right_motor_rotation` @`0x08088815`, ошибка `set right motor speed failed`. Оба колеса подтверждены.

### Низкий уровень
- `fcn.0801f4dc` `✔` — **НЕ «set speed»** (так ошибочно звал прошлый декомпил), а установка бит направления/режима + enable. Вызывается из `*_motor_rotation`.
- `fcn.0807f696` `✔` — установка скважности PWM-канала таймера (`base, ch, duty`).
- Драйвер BLDC-гейта **A4963** по SPI: `a4963_init_v20`, `A4963 init failed` — коммутация железом, конфиг по SPI. Переиспользуем как есть (не переписываем).

### Как это использовать в RC
Наш модуль вызывает `left_motor_rotation(lmdrv_ctx, speed, dir)` и `right_motor_rotation(...)` с нашими значениями скорости 0..6141 → дифф-привод (танковый разворот = разные знаки). Нож — отдельный драйвер (ниже). **Родная коммутация/PWM/таймеры не трогаются** → моторы/драйверы в безопасности.

## 2. Нож (blade) `ⓗ`

Точку входа «задать обороты ножа» + стоп/тормоз — досверить (аналогично `*_motor_rotation`). Для газонокошения нужен вкл/выкл + обороты; для чистого RC можно держать выключенным.

## 3b. Декомпиляция pdg (r2ghidra) — подтверждённое `✔`

Тяжёлый проход `pdg` (движок Ghidra в образе radare2) дал чистый C по ядру:

**`left_motor_rotation` `fcn.0805f620(obj, speed)`** — полностью:
- Объект драйвера `lmdrv` = `*0x2000054c`. Поля: `+8`=режим, `+0x10`=текущее направление, `+2`=последний PWM.
- speed → PWM: `pwm = speed*0x1c20 / K`, где `K∈{0x5fd=1533, 0xbfd=3069, 0x17fd=6141}` по полосам скорости; клампы `0x2ce (718)`, `0x1c20 (7200)`.
- Направление: `fcn.0801f4dc(obj, obj+0x30, dir)`, `dir∈{0,1,2}`.
- Выход PWM: `fcn.0807f696(*0x805fac4 = TIMER 0x40000400, ch=3, pwm)`.

**`fcn.0801f4dc` = запись регистра A4963 по SPI с верификацией** `✔`:
- Пишет биты направления в `*(obj+0xa)` (маска `&0xff8f | 0x10/0x20/0x30`).
- Читает назад и сверяет: `fcn.080840d8`(SPI write) / `fcn.0806caac`(SPI read), до `0x31` ретраев, иначе `-1`. → это интерфейс к мотор-драйверу A4963.

**Батарея/ток `fcn.0802f8c0`** — значения через **vtable BMS-объекта**: `(**(bms+0x18))()`, `+0x1c`, `+0x28`, `+0x2c`, `+0x30` = напряжение/проценты/min/max/ток (питают лог `battery vol=... left motor=%dmA`). Объект BMS = `*(*(unaff_r4+4)+0x18)`.

**Вывод по архитектуре (важно для WP3):** датчики читаются вызовом **метода vtable сервис-объекта** `(**(obj+off))()`, моторы — `*_motor_rotation(obj,speed)`. Наш модуль должен держать указатели на эти сервис-объекты (фикс. RAM, напр. `lmdrv @0x2000054c`) и звать их методы — это переживает пересборку/версию.

> Честно: точные старты функций, доступных ТОЛЬКО через vtable (`right_motor_rotation`, геттеры IMU/одометрии), авто-анализ не ограничивает; их надо доставать **трассировкой регистрации vtable** в конструкторе сервиса (напр. `new xxx_srv`), а не угадыванием адреса (pdg по мид-адресу даёт мусор). Это следующий систематический шаг.

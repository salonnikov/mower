# Дорожная карта и критерии готовности

Цель — не «надстройка-кнопкодав», а **пересадка мозга**: ESP32-S3 сам читает провод
(matched filter) и сам рулит моторами, лечит кручение на месте и промах мимо базы.
Алгоритмы — порт Ardumower Azurit (см. [06-open-source-landscape.md](06-open-source-landscape.md)).
Каждая фаза → проверяемый результат. Безопасность (E-stop) — раньше движения.

## Фаза 0.5 — «Пассажир-разведчик» (неинвазивно) ▶️ ТЕКУЩАЯ
**Делаем:** автономный модуль на ESP32 сверху косилки (IMU + доплер-радар +
катушка-антенна + SD), косилку НЕ вскрываем. Логируем родной сигнал провода,
реальную скорость, поведение у границы и эпизоды «кручения».
**Готово, когда:** есть данные, по которым ясно — кодированный ли родной сигнал
(менять ли sender), какова модель движения, подтверждена причина «кручения».
Дет.: [07-recon-passenger.md](07-recon-passenger.md). Прошивка: `firmware/recon/`.
**Риск:** нулевой (в электронику не вмешиваемся).

## Фаза 0 — Реверс платы (после разведчика)
**Делаем:** фото, трассировка, чипы, **уровни логики**, входы драйверов моторов
(PWM/DIR), аналоговый сигнал катушки, что гонит штатный sender (осциллограф), датчики.
**Готово, когда:** [`hardware/signal-map.md`](../hardware/signal-map.md) заполнен;
ясно — можем ли управлять штатными драйверами напрямую и кодированный ли sender.
Дет.: [00-reverse-engineering.md](00-reverse-engineering.md).

## Фаза 1 — Телеметрия (read-only, нулевой риск)
**Делаем:** ESP32 питается от робота, читает VBAT/ток (INA226), датчики; MQTT → HA.
Быстрый стенд — `firmware/esphome/mower.yaml`.
**Готово, когда:** в HA видно АКБ/токи/датчики, LWT работает.

## Фаза 2 — Приём провода своим matched filter
**Делаем:** ОУ-фронтенд на катушку → ADC-DMA ESP32-S3 → корреляция → signed magnitude.
Если штатный sender некодированный — ставим свой кодированный (база).
**Готово, когда:** в реальном времени видим знак (внутри/снаружи) и дистанцию до
провода, устойчиво в шуме моторов. Дет.: [05-control-and-navigation.md](05-control-and-navigation.md) §A.
**Это — фундамент фикса «кручения на месте».**

## Фаза 3 — Управление моторами + E-stop
**Делаем:** ESP32 рулит входами драйверов (PWM/DIR), PID скорости колёс, нож;
локальный E-stop (lift/tilt/overcurrent) на Core0.
**Готово, когда:** на стенде (колёса вывешены) едет/тормозит по PID; **E-stop ножа
гарантированно срабатывает локально** до любых полевых тестов.
**Риск:** высокий — сначала E-stop, потом движение.

## Фаза 4 — Поведение: кошение + следование по проводу
**Делаем:** конечный автомат MOW/FOLLOW; PID удержания дистанции до провода.
**Готово, когда:** робот косит зону и **плавно следует вдоль провода без кручения**.

## Фаза 5 — Точная стыковка с базой
**Делаем:** возврат по проводу (PID) + одометрия/IMU на финише + детект напряжения
контактов + wiggle.
**Готово, когда:** **стыковка повторяемо без промаха** (не 2–5 см). Дет.: [05 §B](05-control-and-navigation.md).

## Фаза 6 — GPS: трек, карта покрытия, антиугон
**Делаем:** NEO-M8N (UART); трек; гео-забор + lift-угон (локально); сервер строит
карту покрытия (идеи планировщика — из OpenMower), push при тревоге.
**Готово, когда:** трек на карте в HA; вынос/подъём → push + E-stop + сирена.
Дет.: [03-navigation-gps.md](03-navigation-gps.md).

## Фаза 7 — Авто-расписание по погоде
**Делаем:** HA-автоматизация: косить по плану, пропускать дождь/мокро/ночь,
учитывать прогноз и «сколько дней не косили».
**Готово, когда:** робот сам выезжает и сам пропускает дождливые окна; override работает.

## Фаза 8 — Свой бэкенд + PWA (параллельно HA, оба)
**Делаем:** MQTT-ingest → TimescaleDB, REST/WebSocket API, PWA (карта Leaflet + графики),
аналитика износа ножа/АКБ, OTA-раскатка.
**Готово, когда:** телеметрия/команды/карта работают и через HA, и через свой сервис.

## Принципы по всем фазам
- **Обратимость:** все отводы — через разъёмы, штатную плату можно вернуть.
- **Fail-safe:** нет сигнала от ESP32 → нож выключен.
- **Offline-first:** безопасность и кошение живут на Core0, без Wi-Fi/сервера.
- **Один контракт MQTT** ([01-architecture.md](01-architecture.md)) — переносимость на свой бэкенд.
- **Переиспользуем опенсорс** (Azurit/OpenMower), не пишем с нуля; чтим GPL.

# Фаза 1 — пассивный сбор данных навигации (live SWD-поллинг chip1)

Цель: доказать локализацию ДО любых патчей. Малина (RPi2, `192.168.10.81`, `pi/raspberry`) по SWD к **chip1 (гребёнка B)** читает RAM **без halt** (~20–50 Гц), пишет CSV. Косилку гоняем руками/в кошении — собираем `ls/rs` + одометрию + курс + **собственную позу x/y chip1**.

## Карта RAM-телеметрии chip1 (GD32F305, дамп v1)
Указатели защёлкиваются ОДИН раз после старта (в дампе они пустые — образ снят до init), потом полим фикс-смещения.

| Поток | Адрес (разыменование) | Тип | Писатель |
|---|---|---|---|
| **поза x** ⭐ | `*(0x20000090)+0x40` | f64 | DR-интегратор `fcn.0801f70a` |
| **поза y** ⭐ | `*(0x20000090)+0x48` | f64 | `fcn.0801f70a` |
| дистанция (мм) | `*(0x20000090)+0x34` | u32 | `fcn.0801f70a` |
| курс-копия | `*(0x20000090)+0x5e` | s16° | DR |
| **курс (авторит.)** | `*(0x2000039c)+0x18` | s16° | MEMS/attitude (`fcn.08028c70`) |
| **ls** (лев.коил) | `*(*(0x200003a8)+4)+0x0c` | s32 | `receive_new_border_message_callback fcn.0806d0d4` |
| **rs** (прав.коил) | `…+0x14` | s32 | `fcn.0806d0d4` |
| sls/srs | `…+0x24 / +0x28` | s32 | `fcn.0806d0d4` |
| lf (line-found) | `…+0x1c` | s16 | `fcn.0806d0d4` |
| состояние | `0x200000bc` (прямой static) | u32 0..10 | `set_main_state fcn.0807976c` |
| bat_per | live-state `[r4+0x22]` (цепочка `0x200003c4`) | s32 | BMS |

Цепочки: `pose=*(0x20000090)`; `head=*(0x2000039c)`; `bd=*(*(0x200003a8)+4)`. Валидны 0x20000000..0x20020000.

## Инструмент (на малине, `~/mower-swd/`, копия в репо `tools/rpi-swd/`)
- `poll.cfg` — openocd persistent: SWD к chip1, `init`, без halt/shutdown → telnet-сервер 4444.
- `poll.py` — питон: телнет к 4444, защёлкивает указатели, читает блоки (`mdw`), пишет `capture.csv` (t,state,x,y,heading,dist,ls,rs,sls,srs,lf), ~30 Гц.
- `run-capture.sh N` — поднимает openocd в фон, гоняет питон N сек, гасит openocd.

## Порядок
1. SWD малины на **гребёнке B** (DIO→пин18, CLK→пин22, GND→пин6). Косилку **включить** (chip1 под питанием).
2. Залить 3 файла на малину (когда `.81` будет доступна по WG).
3. `bash ~/mower-swd/run-capture.sh 180` — самотест защёлкивает указатели (печатает pose/head/bd ptr и state); если указатели в диапазоне 0x2000xxxx — пишет CSV.
4. Пока пишет — погонять косилку руками/в кошении по разным местам двора.
5. Забрать `capture.csv` на сервер, оффлайн построить карту провода + EKF, проверить точность позы.

⚠️ Риск: если `mdw` без halt вернёт мусор/пусто — значит фон-чтение не идёт; тогда либо краткий halt на сэмпл (chip1 halt терпит, но рвёт тайминги), либо мини-патч chip1 на вывод телеметрии в UART. Сначала пробуем без halt.

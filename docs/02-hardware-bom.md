### GPS: обычный vs RTK
- **Сейчас (антиугон + трек + грубая карта):** NEO-M8N/M9N, точность ~2–5 м. Дёшево, хватает.
- **«Максимально умно» (кошение по карте без провода) — позже:** RTK-пара
  (например **u-blox ZED-F9P** ровер + база, либо приём NTRIP-коррекций).
  Точность ~1–3 см, но дороже и требует открытого неба. На фазе 3 ставим обычный,
  RTK — отдельный апгрейд в фазе 5+. См. [03-navigation-gps.md](03-navigation-gps.md).

## 2. Питание ESP32 от робота

```
VBAT (~20 В, 5S) ──┬── штатная плата (не трогаем)
                   │
                   └── MP1584 (20→5 В) ──┬── 5V → ESP32 (VIN/5V)
                                         └── 5V → GPS-модуль (если 5 В)
                                                 (или 3.3 В с LDO ESP32)
```
- DC-DC обязателен: 20 В напрямую на ESP32 = смерть платы.
- Добавить предохранитель (0.5–1 А) и TVS-диод на вход DC-DC (защита от выбросов моторов).
- Общий GND с штатной платой — **обязательно** (иначе сигналы «висят»).
- Развязать питание: 100–470 мкФ на входе ESP32, керамика 0.1 мкФ у выводов.

## 3. Подключение сигналов (companion)

### Только чтение (безопасно):
- VBAT → делитель 100k/20k → ADC ESP32 (или, лучше, INA226 по I²C — точнее и без делителя).
- Сигнал периметра «inside/outside» → берём с **выхода компаратора** штатного приёмника
  (цифровой), при 5 В — через делитель/level-shifter, на GPIO ESP32.
- Датчики (rain/lift/tilt/bump) → с соответствующих линий MCU, через согласование уровней.
- Холлы одометрии → GPIO с прерываниями (счёт оборотов колёс = пройденный путь).

### Управление (перехват драйверов, не кнопки):
- ESP32 подаёт PWM/DIR прямо на **входы силовых драйверов** моторов (по карте фазы 0),
  через level-shifter если логика драйверов 5 В. Штатный MCU от этих входов отключаем.
- Аналоговый сигнал катушки провода → ОУ-фронтенд → ADC (DMA) → matched filter.
- Если штатными драйверами управлять нельзя (входы недоступны/завязаны на MCU) —
  ставим свой драйвер (BTS7960/DRV8871) на моторы, штатный силовой каскад в обход.

## 3a. Карта пинов — DOIT ESP32 DevKit V1, 30 пинов (наша плата)

Плата пользователя: **ESP32 WROOM-32, CP2102, Type-C, 30-pin DOIT DevKit V1**.
Выведено 25 GPIO; все ADC1-пины на месте. Засады classic ESP32:
- **Аналог — только ADC1: GPIO 36, 39, 34, 35, 32, 33.** ADC2 НЕ работает при Wi-Fi.
- **GPIO 34/35/36/39 — input-only** (нет выхода/подтяжки): только входы/АЦП.
- **Strapping GPIO 0/2/12/15/5** — не занимать критичным на старте.
- GPIO 6–11 заняты flash (не выведены).

Распиновка платы (силкскрин, USB снизу):
- Левый ряд:  EN, 36(VP), 39(VN), 34, 35, 32, 33, 25, 26, 27, 14, 12*, 13, GND, VIN
- Правый ряд: 23, 22, TX0(1), RX0(3), 21, 19, 18, 5*, 17, 16, 4, 2*, 15*, GND, 3V3
  (`*` = strapping)

Финальное распределение (без конфликтов и без strapping-пинов на критике):

## 4. E-stop ножа — самое важное по безопасности

Принцип **fail-safe**: при любой неисправности/потере питания ESP32 нож должен ВЫКЛючаться.

Варианты (по возрастанию надёжности):
1. ESP32 управляет линией **enable** драйвера ножа (если есть): low = стоп.
2. **Low-side MOSFET** в цепи питания драйвера ножа, затвор подтянут к GND (выкл по умолчанию),
   ESP32 открывает его только когда «всё ок». Нет сигнала → нож обесточен.
3. Реле в разрыв питания мотора ножа (нормально-разомкнутое).

Локальные условия немедленного E-stop (на самом ESP32, без сервера):
- lift = 1 (подняли робот),
- tilt > порога (перевернули/сильный наклон),
- bump длительный (застрял),
- ток ножа > порога (заклинило) — по INA226/шунту,
- watchdog: завис основной цикл.

> Штатная плата MI 302 уже имеет свою защиту по lift/tilt — мы её **не отключаем**,
> а **дублируем** на ESP32. Двойная защита, а не замена.

## 5. Сборка и эксплуатация

- ESP32 + DC-DC + GPS — на отдельной платке, залить conformal coating.
- GPS-антенну — наверх корпуса, открытое небо, подальше от моторов (помехи).
- Все отводы с штатной платы — через разъём (чтобы можно было всё вернуть как было).
- Промаркировать каждый провод по `signal-map.md`.
- Тест по шагам: (1) только питание ESP32, (2) +чтение датчиков, (3) +GPS,
  (4) +E-stop (проверить отсечку!), (5) только потом — команды движения.

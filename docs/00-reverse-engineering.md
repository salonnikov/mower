# Фаза 0 — Реверс-инжиниринг платы VILLARTEC MI 302

Цель: получить **карту сигналов** (de-facto «схему») реальной платы, чтобы
безопасно подключить ESP32 — сначала на чтение, потом на управление.
Официального Schaltplan нет, поэтому снимаем его сами.

Результат фазы → заполненный [`../hardware/signal-map.md`](../hardware/signal-map.md).

---

## 0. Безопасность ПЕРЕД вскрытием

- Извлечь ключ безопасности / отключить и **физически вынуть батарею**.
- Нож не трогать руками — даже обесточенный; работать в перчатках.
- Li-ion 5S (≈18–21 В): не закорачивать, не паять под напряжением.
- Сфотографировать всё ДО разборки (расположение разъёмов, цвета проводов).

## 1. Что мы ожидаем увидеть (типовая топология OEM-робота)

Почти все такие роботы построены по одной схеме из 4 узлов:

```
          ┌──────────────────────────────────────────────┐
          │                  MCU (главный)                │
          │   обычно: GD32 / STM32 / Nuvoton / 8051-clone │
          └───┬──────────┬───────────┬──────────┬─────────┘
              │          │           │          │
   ┌──────────▼──┐  ┌────▼─────┐  ┌──▼───────┐  ┌▼────────────┐
   │ Драйверы    │  │ Приёмник │  │ Датчики  │  │ Питание/BMS │
   │ моторов     │  │ периметра│  │ дождь,   │  │ 20V→5V→3.3V │
   │ L/R колёса  │  │ катушки  │  │ lift,    │  │ измерение   │
   │ + нож       │  │ + ОУ/    │  │ tilt,    │  │ I/U батареи │
   │ (H-мост)    │  │ компар.  │  │ bump,    │  │ заряд station│
   └─────────────┘  └──────────┘  │ Холлы    │  └─────────────┘
                                   └──────────┘
```

Узлы, которые надо найти и подписать:

1. **Драйверы моторов** — ищем H-мосты. Маркировки-подсказки: любые силовые
   корпуса SO-8/HSOP/QFN рядом с разъёмами моторов и толстыми дорожками.
   Частые чипы: `A4950`, `DRV8870`, `MX1508`, `TB6612`, дискретные MOSFET-полумосты.
   У каждого мотора 2 силовых провода (+ датчики Холла у BLDC — ещё 5 проводов).
2. **Приёмник периметра** — две (иногда одна) катушки индуктивности спереди,
   идут к операционному усилителю / компаратору (`LM358`, `LM393`, `MCP6002`).
   Это синусоида/импульсы ~ единицы–десятки кГц. Ключевой сигнал для «вижу провод».
3. **Сенсоры** — дождь (две дорожки-гребёнки + компаратор), lift/tilt (микрик или
   акселерометр/ртутный/шариковый), bump (микрики на бампере), Холлы одометрии колёс.
4. **Питание/BMS** — вход 20 В от батареи, DC-DC (часто `MP2307`/`MP1584` → 5 В,
   затем LDO `AMS1117-3.3`). Контакты зарядной станции, шунт/ОУ для измерения тока.

## 2. Инструменты

- Макро-камера / телефон с макро + хорошим светом (или USB-микроскоп).
- Мультиметр с прозвонкой (continuity) и режимом диодов.
- Осциллограф (желательно) или хотя бы логический анализатор (8 каналов, ~$10).
- Пинцет, флюс/припой, тонкие провода МГТФ для отводов.
- Изопропанол (смыть лак/грязь с дорожек для трассировки).

## 3. Пошаговая методика снятия «схемы»

### Шаг 1. Фотофиксация (обязательно)
- [ ] Топ-сайд платы целиком, резко, при боковом свете.
- [ ] Ботом-сайд целиком.
- [ ] Макро каждого IC так, чтобы **читалась маркировка** (строки на корпусе).
- [ ] Все разъёмы крупно + подписать, что куда идёт (мотор L/R/нож/станция/датчики).
- Сложить в `hardware/photos/` (создать папку; в git не коммитим тяжёлое — `.gitignore`).

### Шаг 2. Идентификация чипов
Для каждой надписи на корпусе — поиск даташита (часто верх строки = партномер,
низ = дата/lot). Заносим в `signal-map.md` таблицу: `ref | маркировка | что это | даташит`.
Если маркировка стёрта/«blob» (эпоксидная капля на главном MCU) — определяем по
обвязке (кварц, кол-во ног, питание) и по поведению.

### Шаг 3. Трассировка питания (сначала «землю и плюсы»)
- Прозвонить GND-полигон, найти все земли (общая опора).
- Найти узлы: `VBAT (~20 В)`, `+5V`, `+3V3`. Подписать на фото.
- Это даёт опорные точки и сразу страхует от ошибок подключения ESP32.

### Шаг 4. Трассировка сигналов к MCU
Цель — найти ноги MCU, отвечающие за:
- PWM/направление каждого мотора (к драйверам),
- вход «сигнал периметра» (с выхода компаратора приёмника),
- входы датчиков (дождь/lift/tilt/bump/Холлы),
- измерение U/I батареи (АЦП),
- UART (TX/RX) — если есть отладочный, это джекпот.

Метод: щупом прозвонки от ноги разъёма/датчика до ноги MCU (плата обесточена).
Каждую найденную связь — в таблицу сигналов.

### Шаг 5. Снятие «живых» сигналов (под питанием, осторожно)
С батареей/лабораторником на 20 В и осциллографом/анализатором:
- [ ] Уровни логики MCU: **3.3 В или 5 В?** (критично для согласования с ESP32!)
- [ ] Форма и частота сигнала периметра (для будущего собственного приёмника/декодера).
- [ ] PWM-сигналы моторов: частота, скважность, как кодируется направление.
- [ ] Полярность/уровни датчиков (активный 0 или 1, подтяжки).
- [ ] Есть ли UART-лог на старте (подключить USB-TTL, 115200/57600/9600).

### Шаг 6. Свести в карту сигналов
Заполнить [`../hardware/signal-map.md`](../hardware/signal-map.md). Это и есть наша «схема».
По ней в фазе 5 решаем: **companion** (тапим сигналы) или **replace** (выкидываем MCU).

## 4. Критерий завершения фазы 0

Фаза 0 закрыта, когда в `signal-map.md`:
- определены уровни логики (3.3/5 В),
- найдены и подписаны: моторы (PWM/DIR), сигнал периметра, все датчики, ADC АКБ,
- понятно, есть ли UART,
- сделан вывод: companion vs replace + почему.

## 5. Чего НЕ делать

- Не подключать GPIO ESP32 (3.3 В) напрямую к 5 В-логике — сжечь вход.
  Нужны делители/level-shifter (см. [02-hardware-bom.md](02-hardware-bom.md)).
- Не запитывать ESP32 от 20 В напрямую — только через DC-DC.
- Не управлять ножом из ESP32, пока не отлажена аварийная отсечка (E-stop).

# chip2 — сбор АЦП / семплинг катушек

Механический вывод r2ghidra (`pdg`) — **доверяем**. Строки `⚑` — интерпретация (`ⓗ`).

## Конфиг двойного АЦП — `fcn.08019bf4`

`⚑ ⓗ` Настройка синхронного двойного АЦП под две носовые катушки. `ⓗ`-статус потому что имя elog не привязано; но аргументы вызовов однозначно кодируют каналы и sample-time.

**Доказательства (`✔`) из литерального пула:**
- `*0x8019cf0 = 0x40012400` → **ADC0** (база GD32F30x ADC0). `✔`
- `*0x8019cf4 = 0x40012800` → **ADC1**. `✔`
- `fcn.08012a04(ADC0, 0, 5, 7)` → **регуляр-канал 5, sample time 7** (LEFT). `✔`
- `fcn.08012a04(ADC1, 0, 9, 7)` → **регуляр-канал 9, sample time 7** (RIGHT). `✔`
- `*(ADC0+8) |= 0x100` в конце → взвод бита (`ⓗ` вероятно DMA/внешний триггер enable).

`⚑ ⓗ` Соответствие сторон LEFT=ch5 / RIGHT=ch9 — из контекстного документа; в этой функции подтверждены **номера каналов и sample time**, но не физическая сторона (её надо ловить по разъёму J9 / другим ссылкам).

```c
void fcn.08019bf4(void)   // ⓗ adc_dual_init
{
    int  adc0 = *0x8019cf0;         // 0x40012400  ✔ ADC0
    fcn.08012aa2(adc0, 0x100);      // ⓗ reset/mode
    fcn.08012aa2(adc0, 2, 0);
    uint adc1 = *0x8019cf4;         // 0x40012800  ✔ ADC1
    fcn.08012aa2(adc1, 0x100, 1);
    fcn.08012aa2(adc1, 2, 0);
    fcn.08012900(adc0, 1, 0x60000); // ⓗ mode/sync
    fcn.08012900(adc1, 1, 0xe0000);
    fcn.08012900(adc0, 2, 0x7000);
    fcn.0801285e(adc0, 0);
    fcn.0801285e(adc1, 0);
    fcn.080129ec(0x20000);          // ⓗ external trigger source
    fcn.08012828(adc0, 1);
    fcn.08012828(adc1, 1);
    fcn.08012828(adc0, 2, 1);
    fcn.08012a04(adc0, 0, 5, 7);    // ✔ ADC0 regular ch5, sample-time 7
    fcn.08012a04(adc1, 0, 9, 7);    // ✔ ADC1 regular ch9, sample-time 7
    fcn.080128ca(adc0, 1);
    fcn.080128ca(adc1, 1);
    fcn.0801292e(adc0, 0, 7);
    fcn.080128ca(adc0, 2, 1);
    fcn.080128ba(adc0);
    fcn.0801b65c(1);
    fcn.0801280a(adc0);
    fcn.080128ba(adc1);
    fcn.0801b65c(1);
    fcn.0801280a(adc1);
    *(adc0 + 8) = *(adc0 + 8) | 0x100;   // ⓗ enable bit
}
```

## Запуск цикла сбора — `fcn.08019f84` = `start_collect_data`  (`✔` elog-тег)

`⚑ ⓗ` Аллоцирует приёмный буфер под `[obj+0x92]` отсчётов (размер в байтах = `count<<2`), сохраняет указатель в `[obj+0x70]`, дёргает DMA-старт через vtable объекта катушки (`0x20000078`).

**Доказательства:**
- Объект коллектора: `*0x8019ffc = 0x20000078` `✔`.
- Буфер: `iVar3 = malloc([obj+0x92] << 2)`, `*(obj+0x70) = iVar3` `✔` (размер-в-отсчётах живёт в поле `+0x92`; в контекстном доке = `0x320` = 800 — перепроверить чтением RAM/значения поля).
- `**0x801a054 = 0x10000000; DSB; ISB;` — `ⓗ` взвод отложенного прерывания/софт-триггера (`0x10000000` = бит в неком регистре).

```c
void fcn.08019f84(void)   // ✔ start_collect_data
{
    int obj = *0x8019ffc;                       // 0x20000078  ✔ coil/collect state
    if (*(*(obj + 4) + 0x70) == 0) {            // буфер ещё не выделен
        int buf = malloc(*(*(obj+4)+0x92) << 2);// ✔ count<<2 байт
        *(*(obj+4) + 0x70) = buf;
        if (buf == 0) log_error(...);           // "..\...:0xece"
        else (*(*(*(obj+4)+0x74)+4))(buf, *(obj+4)+0x92);  // ⓗ DMA config(buf,count)
    }
    fcn.0801ba58();                             // ⓗ timer/trigger arm
    (*(*(*(obj+4)+0x74)+8))();                  // ⓗ DMA/ADC start
    *(*(obj+4)+0x80) = timer_now();             // ⓗ timestamp
    if (fcn.0801c818() == 0) { **0x801a054 = 0x10000000; DSB; ISB; }  // ⓗ soft-trigger
    (*(obj+4))[0xac]++;                          // ⓗ счётчик циклов
}
```

## Калибровка базовой линии — `fcn.0801a1f8`  (`✔` строка `base voltage=%d`)

`⚑` Усредняет **8** отсчётов АЦП, делит на 8 (`x*0x2000>>0x10` = `x/8`), пишет результат в **`0x2000004c`** (`✔` — `*0x801a250 = 0x2000004c`). Возвращает признак «база в допуске»: `(base - 0x772) < 0x11d`, т.е. диапазон **`0x772..0x88E` = 1906..2189** вокруг 2048 (`✔`; вне диапазона — лог `base voltage =%d error, set default 2048`).

```c
bool fcn.0801a1f8(void)   // ✔ baseline calib
{
    int adc = *0x801a24c;          // 0x40012400  ✔ ADC0
    int acc = 0; uint8_t n = 0;
    int *field = *0x801a24c + 0x3c;
    do {
        fcn.08012a88(adc, 2);                       // ⓗ start conv
        do { } while (!fcn.08012920(adc,2) && !fcn.08012920(adc,4)); // ждать EOC (busy-wait!)
        n++;
        acc += *field;                              // ⓗ читает результат
        if (n > 7) {                                // после 8 отсчётов
            **0x801a250 = acc * 0x2000 >> 0x10;     // ✔ 0x2000004c = acc/8
            return (acc*0x2000>>0x10) - 0x772 < 0x11d;  // ✔ 1906..2189
        }
    } while (true);
}
```

`⚑ ⓗ` Busy-wait по EOC-биту — одна из причин фолта на SWD-halt (реалтайм рвётся при остановке ядра).

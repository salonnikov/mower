Вывод `pdg` — доверяем. `⚑` = интерпретация (`ⓗ`). Детект **временной** (сегменты по нулю), не FFT.

## Сегментация по нулю — `fcn.08015dcc`

`⚑ ⓗ` Проходит буфер `int16` отсчётов, режет на сегменты по сменам знака; при завершении сегмента зовёт `fcn.08015c68` (фичи). Допуск склейки зазора — **`0x32` = 50** (`✔`: `*** + 0x32 < uStack_3c` перед принудительным закрытием).

**Доказательства:**
- Читает `int16` из `param_1[uVar6]` (`*(param_1 + uVar6*2)`), сравнивает знак текущего и следующего (`iVar2<1`, `iVar1<1`) `✔`.

- Разрыв > 50 отсчётов → `fcn.0801364c` (сброс/финализация) `ⓗ`.

## Фичи сегмента (ширина + знаковая магнитуда) — `fcn.08015c68`

`⚑` Для сегмента (пара соседних под-сегментов противоположной полярности `[+1]==0` vs `==1`) считает:
- **ширину** `iVar3 = (peakEnd - peakStart) + 1`; проверка **band `iVar3-4 < 0x10`** → ширина **4..19** (`✔`; контекстный док говорил 4..15 — здесь по коду 4..19, доверять коду).
- **пиковую амплитуду** `int16` min/max (`iVar12`/`iVar14`) → знаковая величина в `uStack_40` (`✔`).
- **знак/сторону** `uStack_48` (0 или 1) по тому, у какого под-сегмента `[+1]==1` (`✔`).
- требует, чтобы **≥ половины** отсчётов сегмента были ненулевые: `iVar3/2 <= uVar11` (`✔`).
- результат кладёт в список фич `fcn.0801499e(feat, &struct)` (`✔`).

`⚑ ⓗ` Это и есть источник «знаковой непрерывной магнитуды» = поля `str` (амплитуда) + `area` (ширина), которые дальше уедут как `ls`/`rs`. Монотонность по расстоянию до провода и знак=сторона — **гипотеза из физики/контекста**, в коде видно только «signed peak + width».

# chip2 — карта памяти (RAM / периферия)

MCU: **GD32F30x** (Cortex-M4, FreeRTOS). Прошивка `dist/gd32-mainboard-chip2-dump-v1.bin`, load base `0x08000000`, reset `0x080001b4`.
`✔` = подтверждено (константа в пуле / xref / строка). `ⓗ` = гипотеза.

## Как получены доказательства
- Поиск базовых адресов периферии: `r2 ... -c '/v4 0x40004800'` и т.п. (`/v4` — поиск 32-бит значения в образе). `✔` = константа реально присутствует в литеральном пуле и грузится `ldr`-ом.
- RAM-объекты — из разыменования указателей в пулах (`pxw`), см. столбец «доказательство».

## Периферия (APB/AHB), GD32F30x

| Блок | База | Где в образе (пул / site) | Достоверность | Роль (⚑ = ⓗ) |
|---|---|---|---|---|
| **ADC0** | `0x40012400` | пул `0x08019cf0`, `0x0801a24c` | `✔` | Катушка **LEFT**, регуляр-канал **5**, sample-time **7** (`fcn.08019bf4`). Калибровка базы читает ADC0 напрямую (`fcn.0801a1f8`). |
| **ADC1** | `0x40012800` | пул `0x08019cf4` | `✔` | Катушка **RIGHT**, регуляр-канал **9**, sample-time **7**. Синхронный parallel с ADC0. |
| **TIMER1** | `0x40000000` | пул `0x08019f7c` (site `0x8019f32`, в `start_collect_data`) | `✔` | ⚑ Триггер запуска АЦП (TRGO). Точные PSC/ARR в этом заходе из RAM не сняты (контекст-док: PSC=0, ARR=0xC7 — `ⓗ`, перепроверить). |
| **DMA0** | `0x40020000` | пул `0x080199b7`, `0x0801aab9` | `✔` | ⚑ Льёт `ADC0_RDATA` в буфер отсчётов (`start_collect_data` выделяет `count<<2` байт, указатель в `[obj+0x70]`). |
| **USART2** | `0x40004800` | пул `0x08019b28` (site `0x8019a46`, в регионе `process_comm`), `0x080175e0`, `0x08017864`, `0x0801b11c` | `✔` | Транспорт border-JSON в chip1 (`send_border_message` → драйвер USART2). |
| GPIO (катушки/lift/USART pins) | — | — | `ⓗ` | ⚑ Пины катушек, датчиков подъёма и USART2 TX/RX. Точные порты/пины не сняты — задача. |

`⚑ ⓗ` Соответствие LEFT↔ADC0/ch5 и RIGHT↔ADC1/ch9: **номера каналов и sample-time подтверждены кодом** (`fcn.08019bf4`), физическая сторона (какая катушка на каком канале, разъём J9) — из контекст-дока, требует проверки по схеме/GPIO.

## RAM-объекты (SRAM `0x20000000+`)

| Адрес | Что | Доказательство | Достоверность |
|---|---|---|---|
| `0x2000004c` | **base voltage** (калиброванная базовая линия АЦП, среднее 8 отсчётов) | `*0x801a250 = 0x2000004c`, пишется в `fcn.0801a1f8`; строки `base voltage=%d`, `base voltage =%d error, set default 2048` | `✔` |
| `0x20000070` | **объект process_comm** (состояние катушек/lift для JSON; источник `lf`) | `*0x801995c = 0x20000070` (в `send_border_message`); `*0x0801984c = 0x20000070` (источник ключа `lf`) | `✔` |
| `0x20000078` | **объект коллектора/катушек** (буфер отсчётов, счётчики, поле размера) | `*0x8019ffc = 0x20000078` (в `start_collect_data`) | `✔` |
| `[obj+0x70]` | указатель на выделенный буфер отсчётов АЦП | `start_collect_data`: `*(obj+0x70)=malloc(...)` | `✔` |
| `[obj+0x92]` | число отсчётов в буфере (размер = `<<2` байт) | `malloc([obj+0x92]<<2)` | `✔` (значение 800/`0x320` — `ⓗ`, из контекст-дока, снять чтением RAM на живой плате) |
| `[obj+0x80]` | timestamp последнего запуска сбора | `start_collect_data` | `ⓗ` |
| `[obj+0xac]` | счётчик циклов сбора | `start_collect_data`: `[+0xac]++` | `ⓗ` |
| `[+0xe]` (в lift-структуре) | состояние подъёма (0=none,1=left,2=…) | `deal_lift_sensor` (`fcn.08013970`) | `✔` |
| `[+0xf]/[+0x10]/[+0x11]` | дебаунс-счётчики датчиков подъёма (порог 10) | `deal_lift_sensor` | `✔` |

## Flash (регионы, ориентиры)
| Регион | Что |
|---|---|
| `0x080001b4` | reset handler (`✔`) |
| `~0x080159a0–0x08016900` | обработка волны: сегментация/фичи/классификатор/`follow_border_check_wave`/`follow_check_subwave` (`✔`) |
| `~0x080195c4–0x08019c66` | `process_comm.c`: сборка/отправка border-JSON, `start_collect_data`, ADC/семпл-init (`✔`) |
| `~0x0801a1f8` | калибровка базовой линии (`✔`) |
| `~0x0801c8b3–0x0801cb9b` | таблица строк-имён функций (elog-теги) + `..\src\platform\freertos\tasks.c` (`✔`) |

## Открытые задачи (`ⓗ` — снять на живой плате / доп. реверсом)
1. Точные PSC/ARR TIMER1 и частота семплинга (снять из RAM/регистров без halt — иначе фолт).
2. Точный размер буфера отсчётов (`[obj+0x92]`, ожидаем `0x320`=800).
3. GPIO-порты/пины: катушки, датчики подъёма, USART2 TX/RX.
4. Полный обход RX-IRQ USART2 — подтвердить, что callback приёма не подключён (линк односторонний). Строк `cJSON_Parse`/`GetObjectItem` в образе **нет** (`✔`), что косвенно подтверждает.
5. Привязка FreeRTOS-задач (`board_task`/`comm_task`) к точкам входа через `xTaskCreate`.

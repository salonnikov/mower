# chip2 — инвентарь функций (radare2 `afl`)

**Прошивка:** `dist/gd32-mainboard-chip2-dump-v1.bin` (256 КБ, GD32F30x Cortex-M4 Thumb-2, load base `0x08000000`, reset `0x080001b4`).
**Проект r2:** `tools/r2out/chip2.r2proj` — **397 функций** (механический вывод анализатора, доверяем).
**Как получено:** `r2 -q -p tools/r2out/chip2.r2proj -c 'afl'` (проект открывать БЕЗ аргумента-бинаря — с ним r2 молча выходит; байты подтягиваются из карты проекта). Декомпилятор — r2ghidra (`pdg`).

## Обозначения
- `✔` — подтверждено строками/кодом/константами (факт).
- `ⓗ` — интерпретация/гипотеза (что функция делает), выведена из контекста, требует перепроверки.

## Именованные функции (имена из elog-тегов `__FUNCTION__`)
В прошивке есть строки-имена функций (файл-оффсеты `0x1c910…0x1cb9b`), которые логгер elog печатает как тег. Функция, грузящая такую строку через `ldr`, — **и есть** та функция (`✔`, если тег грузится изнутри её границ; `ⓗ`, где r2 слил границы нескольких функций в одну).

| Адрес | Имя (elog-тег) | Достоверность | Что делает |
|---|---|---|---|
| `0x080195c4` | `send_border_message` | ✔ имя/строит JSON | Строит и шлёт cJSON `{"cmd":16,"ls","rs","lf","sls","srs"}` в chip1. Границу r2 слил в `fcn.08019914`. |
| `0x08019914` | (регион `process_comm.c`) | ✔ | Слитый r2-регион: содержит `send_border_message`/`send_seach_border_message`/`send_follow_border_message`/`send_version_message`. pdg частично ломается на VFP. |
| `0x080164bc` | `follow_border_check_wave` | ✔ тег | Конвейер одной катушки: семплы → сегментация → фичи → классификатор. Возвращает код состояния 0..8. |
| `0x08016784` | `follow_check_subwave` | ✔ тег | Вторичный/«search» детект волны (питает `sls`/`srs`). |
| `0x08013970` | `deal_lift_sensor` | ✔ тег | Дебаунс датчиков подъёма (порог 10) → left/both/right/none lifted; состояние в `[+0xe]`. |
| `0x08019f84` | `start_collect_data` | ✔ тег | Аллоцирует буфер `[obj+0x92]<<2`, запускает DMA-сбор АЦП. |
| `0x08013bbc` | `delay_and_start_next_collect_data` | ✔ тег | Планирует следующий цикл сбора. |

> ⚠️ Оговорка по границам: r2 в `afl` дал `send_border_message` под записью `fcn.08019914` (start `0x08019914`, size 850), хотя фактический вход билдера JSON — `0x080195c4`, а `afi.@0x08019624` тоже относит к `fcn.08019914`. Значит r2 слил несколько соседних `send_*`-функций из `process_comm.c` в один регион. Имена-теги надёжны, точные входы бери из этого столбца.

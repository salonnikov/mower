# Живые находки по chip1 (проверено по SWD 01.07.2026)

Всё ниже — с пометками: **✔ проверено на живом чипе замером**, **ⓗ гипотеза (не подтверждено)**. Правило проекта: гипотезы за факт не выдаём.

## Машина состояний
- ✔ `0x200000bc` = state. Наблюдали: **0** = глубокий idle (все сервис-объекты NULL); **2/3** = standby/проснулась (movebase/att/blade объекты выделены, pose ещё NULL); **5** = движение (кошение/езда).
- ✔ **Ошибка E9 = косилка не смогла дойти до ограничительного провода** (навигационная ошибка). НЕ от наших записей в моторы. Возникает штатно, когда она на подставке и не может найти провод.

## Одометрия (энкодеры колёс) ✔
- ✔ Два счётчика колёс, монотонные, симметричные при езде прямо:
  - **левое: `0x2000be28`**, **правое: `0x2000be58`**.
- ✔ Тикают синхронно при движении (за проезд натикали 0→1460+, ~60 тик/с на ходу). Работают даже на вывешенных колёсах (это энкодеры ведущих колёс). Коэффициент тик→мм — ещё не выведен (ⓗ, снять замером).

## Моторы = PWM таймера TIMER2 ✔
- ✔ База **TIMER2 = `0x40000400`**. При езде прямо:
  - **CCR3 `0x4000043c` = левое колесо**, **CCR4 `0x40000440` = правое колесо** (при прямой езде равны, было 4389/4389).
  - CCR2 `0x40000438` = 3927 — **ⓗ вероятно нож**.
  - CCR1 `0x40000434` = 0 (не используется).
  - период CAR `0x4000042c` = `0x1c20` (7200). CNT `0x40000424` свободно бежит.
- ✔ **Запись в CCR3/CCR4 по SWD РАБОТАЕТ и НЕ перетирается контуром**: записали 0 → колёса замедлились/встали, CCR держались 0 (без повторной записи). Ошибку это НЕ вызвало (E9 был про провод).
- ➜ Вывод: **прямое управление колёсами через запись CCR3/CCR4, похоже, реально.** (нужно ещё чисто подтвердить руль: задать CCR3≠CCR4 → колёса по-разному → поворот).

## Что НЕ мотор
- ✔ `0x2000b250` (~1037) — константа, **НЕ команда мотора**. Отброшено.

## Сервис-объекты (указатели; валидны только в активных состояниях)
- ✔ movebase `*(0x20000338)`, IMU/attitude `*(0x2000039c)`, blade `*(0x20000370)`. В idle (state 0) все NULL. pose `*(0x20000090)` — NULL даже в standby, выделяется при движении (ⓗ).

## Коилы / прочее
- ⓗ телеметрия коилов — регион `0x20000740` (+0x24 лев/+0x2c прав) и осцилляция ~9000 у `0x2000b4ac`; сырой ADC-буфер `0x2000fb78+`. С перемычкой видели 2 канала. Точные адреса ls/rs — доснять.
- Батарея: на экране 62% (наблюдение).

## ГЛАВНАЯ ЗАГВОЗДКА для ручного управления
- Моторы под током **только в ходовом состоянии**, а оно же гоняет **border-навигацию** к проводу → на подставке ловит **E9** и, видимо, глушит моторы.
- Нужно: держать **мотор-PWM включённым БЕЗ border-навигации** (не искать провод). Варианты: найти режим/точку, где PWM активен без навигации; ИЛИ обойти/погасить border-проверку; ИЛИ тонкий патч manual-mode. ← это выясняет декомпил.

## Инфраструктура
- Малина от повербанка (стабильна). Сервисы: `mower-openocd` (SWD :4444), `mower-webctl` (пульт **http://192.168.10.81:8080**, телеметрия+STOP). openocd — один telnet-клиент за раз: для захватов веб-пульт останавливать (`sudo systemctl stop mower-webctl`).

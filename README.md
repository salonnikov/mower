# Smart Mower — «умные мозги» для робота-газонокосилки VILLARTEC MI 302

Проект превращает дешёвый OEM-робот с проводом периметра в управляемое,
наблюдаемое и «умное» устройство на базе **ESP32** + серверной части
(**Home Assistant + MQTT** сейчас, собственный бэкенд — потом).

> ⚠️ Дисклеймер. Аппарат содержит вращающийся нож и Li-ion батарею 20 В.
> Любая модификация электроники — на свой риск. Все работы с ножом —
> только при извлечённом ключе безопасности / снятой батарее. Подробности
> по безопасности см. в [docs/02-hardware-bom.md](docs/02-hardware-bom.md).

## Что это за робот

VILLARTEC MI 302 — это **бренд** (продаётся в РФ под маркой VILLARTEC),
а сам аппарат — типовой китайский OEM-робот класса «провод периметра».
Характеристики линейки MI 3xx/4xx:

| Параметр              | Значение                                  |
|-----------------------|-------------------------------------------|
| Навигация             | Провод периметра (boundary wire, ~ кГц)   |
| Ширина кошения        | 160 мм                                    |
| Высота кошения        | 20–60 мм (механика)                       |
| Батарея               | Li-ion **20 В, 2 Ач** (5S)                |
| Моторы                | 2× привод колёс + 1× мотор ножа (BDC/BLDC)|
| Датчики               | дождь, подъём (lift), наклон (tilt), удар (bump 50–60 Н) |
| Управление            | штатное моб. приложение, зарядная станция |

**Официальной электросхемы в открытом доступе нет** — это закрытый OEM.
Поэтому проект начинается с **реверс-инжиниринга реальной платы**:
см. [docs/00-reverse-engineering.md](docs/00-reverse-engineering.md).

## Подход: пересадка мозга, не надстройка

ESP32-S3 становится **реальным мозгом движения** — сам читает провод (matched
filter) и сам рулит моторами. Это лечит конкретные баги MI 302: **кручение на месте**
(непонимание направления провода) и **промах мимо базы на 2–5 см**. Тело робота
(силовые драйверы, датчики, питание) переиспользуем. Алгоритмы берём из опенсорса
(**Ardumower Azurit**), не пишем с нуля. См. [docs/05](docs/05-control-and-navigation.md),
[docs/06](docs/06-open-source-landscape.md).

## Дорожная карта (фазы)

| Фаза | Что делаем | Статус |
|------|------------|--------|
| **0** | Реверс платы: чипы, уровни логики, входы драйверов, сигнал катушки, sender | ▶️ старт |
| **1** | Телеметрия read-only (АКБ/токи/датчики) → HA | планируется |
| **2** | Свой приём провода matched filter (знак+дистанция) → фикс «кручения» | планируется |
| **3** | Управление моторами (PID) + локальный E-stop ножа | планируется |
| **4** | Поведение: кошение + плавное следование по проводу | планируется |
| **5** | Точная стыковка с базой → фикс «промаха» | планируется |
| **6** | GPS: трек, карта покрытия, антиугон | планируется |
| **7** | Авто-расписание по погоде | планируется |
| **8** | Свой бэкенд + PWA (параллельно HA) | опционально |

Подробно: [docs/04-roadmap.md](docs/04-roadmap.md).

## С чего начать прямо сейчас

1. Прочитать [docs/00-reverse-engineering.md](docs/00-reverse-engineering.md).
2. Снять плату, сделать макро-фото обеих сторон (см. чек-лист там же).
3. Заполнять `hardware/signal-map.md` по мере определения сигналов.
4. Подключить к **существующему** Home Assistant — см. [server/SETUP.md](server/SETUP.md)
   (свой HA не разворачиваем; нужен только ESPHome add-on или брокер MQTT).
5. Прошить ESP32 из `firmware/esphome/mower.yaml` (пока в режиме «только датчики»).

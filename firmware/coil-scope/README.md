# coil-scope — «осциллограф из ESP32» для сигнала провода периметра

Цель первого шага (фаза 0.5): **без вскрытия косилки** увидеть родной сигнал
провода — есть ли он, на какой частоте, как выглядит, и **кодированный ли он**.
Осциллографа нет — его роль играет сама ESP32 (веб-морда с формой + спектром).

## Катушка из витой пары (0 ₽)

1. Расплети витую пару, возьми **один** провод (длина 5–10 м).
2. Намотай **~150–200 витков** на цилиндр Ø ~6–8 см (бутылка, скотч, банка).
   Больше витков = больше сигнал. Сердечник не нужен (воздушная катушка ок).
3. Концы катушки — два вывода. Зафиксируй намотку скотчем.

## Подключение к ESP32 (минимум, без деталей)

| Катушка | ESP32 |
|---------|-------|
| вывод A | **GPIO36 (VP)** |
| вывод B | **GND** |

Питание ESP32 — **повербанк по USB**. Больше ничего не нужно для первого теста.

> Внимание: GPIO36 — только вход (ADC1), это правильный пин. Не перепутай с 5В/3.3В.

## Прошивка (плата в Mac, бинарь собран на Linux)

Без домашнего Wi-Fi и без настроек: ESP поднимает свою точку, смотришь прямо с телефона.
Готовый бинарь: `build/coil-scope-full.bin` (один файл, шить по адресу **0x0**).

### Шаг 1. Залить бинарь из браузера (без установок на Mac)
1. Воткни ESP32 в Mac по USB.
2. Открой в **Chrome/Edge**: `https://espressif.github.io/esptool-js/`
3. Baudrate 921600 → **Connect** → выбери порт платы.
4. Add file → `coil-scope-full.bin`, offset **0x0** → **Program**.
   (Если не коннектится — зажми BOOT на плате при подключении.)

### Подключение датчиков
- **Катушка:** один конец → GPIO36 (VP), другой → GND.
- **IMU LSM6DS3:** VIN→3V3, GND→GND, SCL→GPIO22, SDA→GPIO21, **CS→3V3** (режим I2C).
  Остальные пины (SAD/OCS/SCX/SDX/INT) не подключать. На странице покажет
  «IMU (0x69)» если найден, иначе «НЕ НАЙДЕН».

### Шаг 2. Смотреть с телефона (AP-режим)
1. Подключись телефоном к Wi-Fi сети **`coil-scope`** (пароль `mower1234`).
2. Открой в браузере **`http://192.168.4.1/`**.
3. Видно: сигнал провода (форма/спектр/частота) + IMU (наклон, угловая скорость).

> MQTT/Home Assistant в этой версии нет (для поля не нужен). Понадобится логирование
> в HA — добавлю STA-режим (подключение к домашней сети) отдельной сборкой.

## Пересборка (это делаю я, на Linux, в контейнере)
```
cd firmware/coil-scope
docker run --rm -v $PWD:/p -w /p -v /root/.pio-core:/root/.platformio \
  python:3.12-slim bash -lc "pip install -q platformio && pio run"
# затем merge в один файл build/coil-scope-full.bin (см. историю команд)
```

## Как пользоваться

- На странице — **пиковая частота**, **форма** и **спектр** (обновление ~3 раза/с).
- Поднеси катушку к проводу периметра (или к зарядной базе/станции — там sender).
- Води катушкой и поворачивай её: ищем **чёткий пик** в спектре и максимум амплитуды.
- Частота/амплитуда дублируются в MQTT (`mower/mi302/recon/wire_freq`, `/wire_mag`) —
  видно в Home Assistant.

## Что это нам даст

- **Частота** родного сигнала → настроим под неё приём в мозге.
- **Форма/спектр** → понять, простой это тон (один пик) или сложный код (широкий
  спектр/гребёнка). От этого зависит, менять ли sender (docs/06).
- Если без усиления сигнал слаб — добавим смещение Vcc/2 (2 резистора) или
  операционник. Сначала пробуем как есть.
